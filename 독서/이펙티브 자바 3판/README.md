# 기술 용어

## 자바가 지원하는 자료형 : 인터페이스, 클래스, 배열, 기본형 / 인터페이스, 클래스, 배열 = 참조 타입(reference type)

## 어노테이션은 인터페이스의 일종, enum은 클래스의 일종

## 클래스의 인스턴스와 배열은 객체(object)인 반면, 기본형인 값은 그렇지 않다

## 클래스의 멤버 : 필드, 메서드, 멤버 클래스, 멤버 인터페이스

## 메서드 시그니처 : 메서드명 + 매개변수 타입들. 예를 들어 "foo(int a, int b) { // }" 라는 메서드가 있다면 메서드 시그니처는 foo(int a, int b)가 된다.

## 상속을 서브클래싱과 동의어로 사용함. 인터페이스 상속 대신 클래스가 인터페이스를 implements한다 혹은 인터페이스가 다른 인터페이스를 extend한다고 표현한다

## 아무것도 명시되지 않은 접근 수준을 이야기할 때는 package-private을 쓴다

## API : 공개(exported) API의 약자. 개발자가 클래스, 인터페이스, 패키지를 통해 접근할 수 있는 모든 클래스, 인터페이스, 생성자, 멤버, 직렬화된 형태를 말함. 클래스, 인터페이스, 생성자, 멤버, 직렬화된 형태를 총칭해 API 요소라 한다

## API를 쓰는 프로그램 작성자(사람) = API의 사용자(user), API를 사용하는 클래스(코드) = API의 클라이언트
